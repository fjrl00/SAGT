Versión 1.0.206.0.0 Pre-Alpha   Fecha: 13:51 10/07/2012
=======================================================
- Se ha corregido un bug que se producía al ejecutarse el método ReadColumnOmit y no existir una tabla de frecuencias lo que provocaba una NullReferenceException. Se ha corregido comprobando si existe la tabla de frecuencias.
- Se ha corregido un error al leer el formato de las fechas en la base de datos.
- Se ha detectado que se produce errores al leer la fecha cuando la hora solo se compone de un digito.
Por ejemplo: 
string sDateTime = "10/07/2012 09:50:15";
DateTime dateFile = DateTime.ParseExact(sDateTime, "dd/MM/yyyy HH:mm:ss", new CultureInfo("es-ES", true));
Esto provoca una FormatExceptions. Se ha corregido de la siguiente manera:
string[] timeFormats = {"d/MM/yyyy HH:mm:ss", "dd/MM/yyyy HH:mm:ss", "dd/MM/yyyy H:mm:ss"};
DateTime dateFile2 = DateTime.ParseExact(sDateTime, timeFormats,
                                   new CultureInfo("es-ES", true), DateTimeStyles.AssumeLocal);
- Se produce una InvalidCastException al tratar de convertir un null en string esto es devido a que el campo comentario de un fichero de medias esta a null. Se ha corregido comprovando que es distinto de null antes de realizar el casting.
- Se ha comprobado que al ejecutar la aplicación en una máquina con Windows XP en inglés la ventana de configuración de gráficos no muestra los textos correctamente ya que se solapan etiquetas y cuadros de textos.
En consecuencia se ha decidido poner las etiquetas sobre los cuadros en lugar de al lado de estas.
- Cuando se pulsa analizar una suma de cuadrados y luego se selecciona editar facetas se produce una excepción no controlada de tipo Analysis_and_G_StudyException. Esta se produce porque la clase Analysis_and_G_Study.cs no encuentra el nombre del fichero de procediencia.

SAGT Versión 1.0 build 203.0.2 Pre-Alpha
========================================
- Se han corregido los siguientes bugs:

	- Se produce una InvalidCastException en el método DataSet2TableG_Study de la clase TableG_Study_Percent (método DataSet2TableG_Study que devuelve un TableG_Study_Percent)  Esto se producía al intentar convertir en double un campo vacío de un dataRow, se a corregido comprobando si esta vacío antes de realizar la conversión.

Se le pasaba un dataSet con los proyectos con el mismo nombre (este dataSet se crea para comprobar si existe un proyecto con el mismo nombre) en lugar de un dataset con el proyecto que se quiere insertar.

	- Se ha encontrado un error con los números decimales de la tabla de frecuencias. El tipo double al pasarlo a string usa la coma decimal pero para poder ser introducido en la base de datos tiene que usar el punto decimal, porque sino interpretaría que son dos datos separados por una coma en lugar de un número decimal.


SAGT Versión 1.0 build 202.0.2 Pre-Alpha
========================================
- Se ha añadido la posibilidad de especificar el espacio de trabajo. Herramientas --> Configuración--> Pestaña conexión --> Botón Seleccionar espacio de trabajo.
- Se han corregido algunos Bugs:
	- Error al realizar una consulta sobre proyectos en la base de datos: la tabla devuelta siempre se llamaba "Table". Ahora antes de devolver la tabla se corrige introduciendo el nombre correcto.
	- Al crear un proyecto los campos director y fecha no deben ser accesibles.

En castellano se emplea la coma decimal mientras que en inglés se usa el punto decimal. Se ha corregido mediante una conversión del

Con lo cual si luego se intentaba leer un fichero en otra maquina con otra referencia cultural se producía un error en el formato de la fecha. Se ha corregido forzando a que se lean y escriban los archivos  en una sola referencia cultural (Se ha empleado la española). El como se escriban los datos en el acrhivo es independiente de como se mostrarán en pantalla. Será siempre en la referencia cultural del frameWork instalado.
	- Se ha detectado que se puede editar un proyecto poniéndole el nombre de otro proyecto existente, con lo cual tendíamos dos proyectos con el mismo nombre. Esto se impide comprobando que no existen duplicidades antes de guardar los cambios. En caso contrario no se permite guardarlos.

	- Se ha corregido el color del menú proyectos.
