Versión 1.0.205.0.2 Pre-Alpha   Fecha: 10:43 10/07/2012
=======================================================
- Se ha detectado que se produce errores al leer la fecha cuando la hora solo se compone de un digito.
Por ejemplo: 
string sDateTime = "10/07/2012 09:50:15";
DateTime dateFile = DateTime.ParseExact(sDateTime, "dd/MM/yyyy HH:mm:ss", new CultureInfo("es-ES", true));
Esto provoca una FormatExceptions. Se ha corregido de la siguiente manera:
string[] timeFormats = {"d/MM/yyyy HH:mm:ss", "dd/MM/yyyy HH:mm:ss", "dd/MM/yyyy H:mm:ss"};
DateTime dateFile2 = DateTime.ParseExact(sDateTime, timeFormats,
                                   new CultureInfo("es-ES", true), DateTimeStyles.AssumeLocal);
- Se produce una InvalidCastException al tratar de convertir un null en string esto es devido a que el campo comentario de un fichero de medias esta a null. Se ha corregido comprovando que es distinto de null antes de realizar el casting.
- Se ha comprobado que al ejecutar la aplicación en una máquina con Windows XP en inglés la ventana de configuración de gráficos no muestra los textos correctamente ya que se solapan etiquetas y cuadros de textos.
En consecuencia se ha decidido poner las etiquetas sobre los cuadros en lugar de al lado de estas.

 public static Dictionary<string, double?> ImportSsq_File(string pathFile)
que lee los ficheros exportados por EduG de Sumas de cuadrados.
Los ficheros tienen el formato de un fichero separado por comas (un csv) donde el separador es el caracter ';'. El fichero se compone de tres columnas: 1º diseño de
facetam, 2º suma de cuadrados, 3º grado de libertad.
Ejemplo:

P;6,777778;2
O;0,5;2
C;140,111111;2
PO;0,333333;4
PC;0,555556;4
OC;0,333333;4
POC;0,333333;8

public static bool WritingFileExportEduG_Ssq(TableAnalysisOfVariance tbAnalysis , string pathFile)
 
Exporta las sumas de cuadrados para que pueda ser leido por EduG 6.0. El formato es el mismo que el descrito en el apartado anterior.
